配置hexo新手体验过程

Posted on 2023-02-24

配置自己的博客|Hexo博客框架|无坑版|Wdindows版本(参考于CodeSheep)

部署环境

安装Git

  • 官网[下载地址]：(“https://git-scm.com/downloads")
    配置环境变量，右击电脑->属性->环境变量->在path里面添加

    安装node.js

  • 官网[下载地址]：(“https://nodejs.org/en/")下载长期支持版的（左边绿色框）
  • 打开终端，这里一般打开cmd或者也可以使用上一步下载的 Git里面的Git bash(这里以Git Bash演示)
  • 查看node和npm的版本(检测是否安装好)
    node -v
    npm -v

    安装Hexo博客框架

  • 利用淘宝镜像源下载cnpm工具
    npm install -g cnpm –registry=https://registry.npm.taobao.org
    此时cnpm工具就下载好了 输入cnpm -v查看
  • 利用cnpm安装Hexo框架
    cnpm install -g hexo-cli
    下载完成可以输入 hexo -v验证一下

初始化个人博客

  • 在E盘下建立blog文件夹(你的博客的根地址，如果整体部署错误可以删掉这个文件夹重新来过，希望大家不会遇到)
  • 切换到此目录下 cd e:\blog
  • 生成博客(初始化) hexo init(此命令是win版本的命令)
  • 启动博客
    hexo s
    启动成功会提示启动在localhost:4000端口
    crtl + c 推出
    在本地的打开通常是我们自己调试预览的时候使用

    写博客

  • 新建一篇文章
    hexo n “你的文章标题”
    它会自动生成一篇文章 地址是e:\blog\source_posts
  • 生成博客
    1.退回到e:\blog目录 hexo -clean
    2.生成 hexo -g
    3.启动查看 hexo -s 在本地4000端口查看
    注意：
  • 输入命令的当前目录应该是E:\blog
  • 编辑文章方法有很多
    可以使用VSCode等工具

把博客部署到远端

(可以部署到云平台、GitHub等，这里以GitHub演示)

GitHub

  • 登录自己的GitHub，新建仓库 右上角 new repository
  • 在Repository name 输入 lishupeng9.github.io
    !!!一定要输入自己的名字.github.io !!! 然后创建完成

    回到命令行

  • 安装Hexo插件
    cnpm install -save hexo-deployer-git
  • 设置_config.yml文件
    vim _config.yml 在最底部的Deployment，修改它的一血内容
    1.type: ‘git’
    repo: https://github.com/lishupeng9/lishupeng9.github.io.git
    branch: main
    注意： 每个冒号后面要加一个空格！！！
    2.保存退出

    部署到远端

  • hexo d
    注意：输入hexo d命令后报错 failed to push some refs to ‘地址’，此时是因为没有配置git的username和邮箱

    打开git bash 配置 git config –global user.name “username” 这里的username可以自己定义
    git config –global user.email username@qq.com –邮箱不用双引号） 配置完后再进行hexo n 进行创建
  • 弹出登录框输入GitHub的账号密码
  • 此时输入 lishupeng9.github.io就可以远程打开了

    给个人博客换主题

    (大家可以自己搜索博客主题，这里使用一个主题来演示)
  • 下载yilia主题并克隆到自己的博客目录(blog\theme)下(基于blog目录操作)
    git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia
  • 修改_config.yml 文件
    找到#Extensions下面的##themes ，把后面的换成yilia
  • hexo clean
    hexo g
    hexo s
    此时已同步到本地
    推到远端 hexo d 在通过地址就可以访问，但是不会立马生效，需要稍等一下，多刷新几下

（“https://www.cnblogs.com/juzaizai/p/14472119.html”）
